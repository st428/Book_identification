# 后续训练方案

## 当前结论

最近一次识别模型训练已完成，但不能部署到 Web 端：

- 训练集：1246 个 crop
- 验证集：310 个 crop
- `best_accuracy` 独立验证精确命中：0/310
- 平均归一化相似度：0.374
- 常见错误输出集中塌缩成 `K6.3/Z`、`K66.3/Z`、`K669.3/Z`、`D66.3/Z`

主要判断：这次使用很小的自定义字符表后，识别头与 PaddleOCR 预训练参数形状不匹配，识别头基本等于从头训练；当前 1500 级别样本不足以让模型稳定学会完整索书号。训练日志中的 `acc=0` 也说明不能直接采用该 checkpoint。

## 总体路线

先做“识别训练”，检测训练后置。

原因是当前红色标签定位和几何补框已经能支撑大部分 Web 流程，瓶颈更多在 crop 内文字识别、相邻书号互相串字符、长后缀和冒号规则。识别模型稳定后，再考虑专门训练检测模型去解决薄书、双排标签、角度倾斜导致的漏框。

## 数据流程

1. 新照片先放入独立批次目录，例如 `traindata/newdata/` 或按日期新建目录。
2. 用现有 Web/导出脚本跑出 crop 和初始识别结果。
3. 生成带 crop 图的 HTML 复核表，人工修正 `labels_draft.tsv`。
4. 人工确认后的 TSV 用 `import_corrected_review.py` 导入为 `rec_label_export_*_corrected`。
5. 用 `organize_project_data.py` 汇总人工校验数据，自动去重并生成 `training_data/recognition/` 下的标准结构。
6. 用 `prepare_rec_training.py` 生成 PaddleX/PaddleOCR 训练目录和配置。
7. 训练完成后必须用 `evaluate_rec_checkpoint.py` 做独立评估，不能只看训练日志中的 best。

## 标注规则

- 只把 crop 中书号完整、清晰、与标签一致的样本加入训练。
- 书号允许重复；同名/重名索书号是正常情况，不能按唯一键去重标签。
- 冒号 `:`、数字后缀、短横线后缀要保留，例如实际看到 `D669.3/XXW:1` 就按原样标。
- 明显残缺、串入相邻书号、纯白/纯黑、文字和图片不对应的 crop 不进入训练。
- 不完整分类号如 `D66`、`D669.3` 可用于复核或不确定展示，但不要作为强排序训练样本。
- 标签统一大写，避免混入中文空格、全角斜杠、不可见字符。

## 下一版识别训练策略

1. 保留预训练能力
   不再优先使用极小自定义字符表重训识别头。下一轮优先采用 PaddleOCR 原有英文数字/中英文字典或完整字典方案，尽量避免输出层被重置。

2. 关闭不适合索书号的拼接增强
   当前配置里的 `RecConAug` 会把多个文本图拼接训练，这对自然文本可能有帮助，但对短而结构固定的索书号容易制造不真实样本。下一轮先禁用。

3. 调整长度和宽度
   `max_text_length` 建议提高到 32；长书号或带冒号的样本，识别宽度从 `320` 试到 `480` 或 `640`，避免后缀被压扁。

4. 降低学习率并延长训练
   微调优先用较小学习率，例如 `1e-5` 到 `5e-5`，训练 20-40 epoch，使用早停。不要因为 8 epoch 内无精确命中就盲目加大数据。

5. 按照片划分验证集
   验证集必须按原始照片分组切分，避免同一张照片的相似 crop 同时出现在训练和验证里。

6. 双模型过渡
   训练模型先作为候选 OCR 使用：只有当新模型结果通过正则、置信度和排序一致性检查时才替换默认 PaddleOCR；否则回退默认 OCR。

## 验收标准

识别模型进入 Web 端前至少满足：

- 验证集精确命中率达到 80% 以上
- 平均归一化相似度达到 0.95 以上
- 对带冒号、数字后缀、薄书、相邻后缀串扰样本单独抽测通过
- 在 10 张以上未参与训练的整架照片上，排序建议不能明显劣于当前默认 OCR

## 数据量建议

先不要盲目堆新照片。下一步应先修正训练配置，用现有人工校验数据重新跑一版。如果配置修正后验证集仍低于预期，再补数据：

- 优先补薄书、后缀相近、冒号、倾斜、右侧漏检、双排红标签这些失败类型。
- 每类至少 200-300 个干净 crop。
- 按一张书架照片 25-35 个有效 crop 估算，每一类大约需要 8-12 张针对性照片。

## 检测训练启动条件

识别训练稳定后，再开始检测训练。检测训练需要单独标注书号区域或红色标签区域，目标是解决：

- 薄书漏框
- 相邻书号框过宽导致串字符
- 双排红色标签误框
- 拍摄倾斜导致局部漏检

检测训练不要和识别训练混在一起评估，必须分别看框定位质量和 OCR 质量。
